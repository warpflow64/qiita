# 5/13: 会場前日準備の日

この日は会場への機材の運び込みと最終チェックが行われました。
自分は夕方ごろ到着して会場のトラブル対応に当たりました。
そこで一番最初に`hall-sw01`と配下のAPがリンクアップしないというトラブルでした。
### `hall-sw`のトラブル
原因としては、モジュールのケーブル長に対して LAN ケーブルの長さがあっておらず、疎通が取れなかったというもので、
それに対して試行錯誤しました。

まず、コンソールケーブルを持って先輩と急行しました。
そしてコンソールを接続して思ったのですが設定は全て間違えていませんでした。
その代わりに`mtu`系のエラーが色々なAPから出ていました。
この時点で`mtu`の設定がどこかで間違えてしまい`mtu`の自動縮小がうまく動いておらずパケットロスが発生しているということです。
なので`mtu`を手動で固定してみましたが効果はありませんでした。
次に怪しいと思ったのは SFP+ のポートです。
上流の`eps-sw01`とはLANケーブルで 10Gbps で接続されていましたが、これの LAN ケーブル長が 100m を超えているのでリンクダウンしていて
それが原因で通信がうまく流れていないという説です。
そしてそれを検証するべく`Gig0/1`のポートに接続し直接 1Gbps を収容する構成に変更しました。
![hall-sw01](https://raw.githubusercontent.com/warpflow64/qiita/main/img/PXL_20260513_071537338.jpg)
すると`mtu`系のエラーも消え、通信も正常に流れ始めました。
そして更なる検証を進めるべく`Ten0/2`に接続するなどポート不調の可能性も疑いましたが、どれもダメで結局LANケーブル長とモジュールの問題だと結論づけました。
そして会議の時間となり一旦 NOC 部屋に帰りました。

### 学び: NOC 全体会議
各所でのトラブルにそれぞれが分担して当たりましたが、それぞれの結果を集約するべく会議が開かれました。
そして自分の対応した`hall-sw01`の報告を行いました。
その際に自分は、自分のやったことを全て時系列通りに言いました
「最初にmtu系のエラーが出て〜、mtuを固定しましたがうまくいかず〜、モジュールの問題を疑い〜刺すポートを変更して〜次にポートの不調を疑い〜」
そしてそれを書き起こしていた void さんと こうきち さんが「結局何したの？」と言われてしまいました。
そりゃあ確かに、やったことを全て言われたらパンクするし単純にわかりにくいのは当然です。
すると後から応援に来てくださった しののめ さんが自分のやったことと原因究明の一連の流れを要約して伝えてくださりました。
「モジュールとケーブル長が合わずにmtu系のエラーが出ていて今はGigポートに直接収容することで解決しています。」
うーん、すごすぎる。。

この後にも上流の IP アドレスが違うというヤバめのトラブルはありましたがマジでリーダの方を初め NOC チームの人が優秀で普通に乗り切ってました。

ただ、結構な人の量が来ると必ずと言っていいほど起こるのがトラブルでセッションが始まるごとに`eps-sw02`配下のスイッチとの疎通が怪しくなるというトラブルが発生しました。
NOC 部屋から Deadman を監視しているとまじで通信が到達しない場合もあるレベルで不安定になってきました。
**よし、じゃあコンソールケーブルを担いで会場へ Go ！！**
という訳にはいかず人がはけたタイミングでこっそりと作業に取り掛かることになりました。
しかし、スイッチの設定が特段間違っているというわけではないんです。
**わーお!!**
これはやばい展開になってきましたねぇ！ 楽しくなってきましたねぇ！

そしてしばらく検証を進めると、ケーブルの品質が怪しいらしいという(なんやねんそれ)みたいなトラブルでした。
そこで会場に元々あったLANコンセントを使わずに新規で引き直すことに変更、結構な長さのケーブルを担いで eps-sw02 から配下のスイッチまでのLANケーブルを引き直しました。
そんなこんなでありとあらゆる可能性を模索して L3 ~ L0 まで見直しました。

### 学び: STP 入れんなマジ
えーっとですね、大変申し訳ないのですが。`eps-sw02`をいじっていたところ STP の設定を間違って入れてしまい、配下のスイッチ＆ AP との通信を寸断するというやらかしをしました。
NOC 部屋ではアラートが全部付いていよいよ`eps-sw02`が燃えたかという話をしてたそうですwww
すみません。笑い事じゃないですよね
SSH で"勝手に"変更してたのですがコマンドを打った瞬間に SSH の疎通も切れて冷や汗が止まらなかったです。
反省として、コマンドを slack に流して確認を貰えばよかったです。
そしてその後 slack で連絡をしたのですが、めっちゃ NOC 部屋に帰りたくなかった。。
ですが、やばい空気かな、、と思いながら帰ったのですが
めっちゃ暖かく迎えてくださいました。ありがとうございます。
商用ネットワークじゃないんだから、ドンマイドンマイ、と言って流してくださいました。

いろんなことがありつつ1日目は終了しました。本当にお疲れ様でした。
